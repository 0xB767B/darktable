contributing

  • use it! and then discuss your bugreports, feature requests, suggestions, translations, patches .. on the mailing list.
  • post documentation, tutorials, your experiences at the forums.
  • post beta: write image operation plug-ins.
  • translate dt into your language (in case it didn't already happen:
    read the short instructions in the file TRANSLATORS in git.
  • produce helpful stacktraces for bugreports (minimal steps):
    $ ./configure --prefix=/usr --enable-debug
    $ make && sudo make install
    $ gdb darktable
    ..crash dt here..
    (gdb) bt
    and post the backtrace. even better: try a few prints and figure out what seems to be the problem. best: fix it and supply a patch!
  • supply a patch to current, possibly changed master:
    $ git checkout -b yourbranch
    ..make your important changes..
    $ git pull
    $ git rebase master
    $ git format-patch master
  • maybe you want your name and email in your .git/config, so it will appear in dt's log:
    $ git config --global user.name "alfons"
    $ git config --global user.email "dreggn@gmx.at"
