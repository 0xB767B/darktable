there are two principal ways to get darktable: get a stable release or get an up-to-date snapshot via git. pick one:

prerequisites

  • *nix (tested: ubuntu jaunty+karmic, arch linux, gentoo, fedora, mac osx with macports)
  • sqlite3
  • libjpeg, libpng
  • libraw (supplied)
  • gtk+
  • cairo
  • libglade2
  • lcms
  • gegl (optional)
  • exiv2
  • optional: gcc>=4.4 (openmp, earlier versions could not handle openmp statements in non-main threads)

current release for ubuntu amd64

  • get the .deb file darktable_0.2_amd64.deb.
  • dpkg -i darktable_0.2_amd64.deb
  • Applications - Graphics - darktable

current release from source

  • grab the 0.2 source .bz2
  • ./configure [--enable-debug] [--prefix=/usr]
  • make all install
  • darktable

current release for gentoo

  • unfortunately darktable is not in portage, so you have to:
  • download this ebuild
  • mkdir -p /usr/portage/media-gfx/darktable/
  • cp darktable-0.2.ebuild /usr/portage/media-gfx/darktable/
  • ebuild /usr/portage/media-gfx/darktable/darktable-0.2.ebuild digest
  • emerge darktable
  • darktable

current release for arch linux

  • arch linux users will know what to do with this file:
  • download this PKGBUILD
  • or: build the svn head with this AUR version
  • if you run into trouble, check if this related bugreport helps you out.
  • thx to chressie for this.

current release for mac osx

  • download this disk image
  • mount the thing
  • pull the darktable icon into your applications folder
  • it worked on a few macs (100%) so far, good luck :)
  • thx to u3dreal for going through all the pain for this mac bundle!

git version

  • git clone git://darktable.git.sf.net/gitroot/darktable/darktable or
    git pull
  • autoreconf --install
  • ./configure [--enable-debug] [--disable-gegl] [--prefix=$(pwd)/install]
  • make install
  • install/bin/darktable
  • let's rock!
