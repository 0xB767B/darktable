there are two principal ways to get darktable: get a stable release or get an up-to-date snapshot via git. pick one:

prerequisites

  • *nix (tested: ubuntu karmic lucid, arch linux, funtoo, gentoo, fedora, macintosh osx with macports)
  • required packages: sqlite3, libjpeg, libpng, libraw (supplied), gtk+-2, cairo, libglade2, lcms, gegl (optional), exiv2, gconf, tiff
  • optional: gcc>=4.4 (openmp, earlier versions could not handle openmp statements in non-main threads)

micr0soft(r) windoos(tm)

  • unfortunately the community of this commercial distro didn't natively build dt yet
  • but there's a better solution for you to try:
  • download and burn an iso from Pascal's ubuntu remasters.
  • reboot your machine.

ubuntu packages

  • get an appropriate darktable*.deb.
  • or a nightly git package.
  • # dpkg -i darktable*.deb
  • Applications - Graphics - darktable
  • you may also be interested in downloading a few icc profiles from there.

current release from source

  • grab the 0.5 source .bz2
  • $ tar xvjf darktable-0.5.tar.bz2 && cd darktable-0.5
  • $ ./configure [--enable-debug] [--prefix=/usr]
  • $ make
  • # make install
  • $ darktable

funtoo/gentoo linux

  • unfortunately darktable is not in portage, so you have to:
  • download this ebuild for the git version
  • or this ebuild for the 0.5 release
  • create a local portage overlay, if you don't have one already:
  • # echo 'PORTDIR_OVERLAY="${PORTDIR_OVERLAY} /usr/local/portage"' >> /etc/make.conf
  • # mkdir -p /usr/local/portage/media-gfx/darktable/
  • # cp darktable-*.ebuild /usr/local/portage/media-gfx/darktable/
  • # ebuild /usr/local/portage/media-gfx/darktable/darktable-*.ebuild digest
  • # emerge darktable
  • $ darktable

arch linux

  • pacman -S darktable
  • thx to chressie for this, arch is non-ancient :)

current release for mac osx

  • download the 0.4 disk image
  • mount the thing
  • pull the darktable icon into your applications folder
  • it worked on a few macs (100%) so far, good luck :)
  • thx to u3dreal for going through all the pain for this mac bundle!
  • problems with gconf? try:
    sudo launchctl load -w /Library/LaunchAgents/org.freedesktop.dbus-session.plist

git version

  • git clone git://darktable.git.sf.net/gitroot/darktable/darktable or
    git pull
  • $ ./autogen.sh
  • $ ./configure [--enable-debug] [--disable-gegl] [--prefix=/usr]
  • $ make
  • # make install
  • $ darktable
  • let's rock!
