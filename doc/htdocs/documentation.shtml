known issues

  • we found some versions of image magick to cause problems. in particular there was a problem with a funnily linked libpng in gs in turn linked to by image magick on arch linux (thanks to chressie for pointing this out).
  • some openmp implementations seem unstable (as observed with gcc-4.3.2 on gentoo causing image magick to use openmp). try to configure with something like CC=gcc-4.2 CXX=g++-4.2 ./configure if this doesn't work, re-emerge image magick without openmp support :( (i'm working on it...)
